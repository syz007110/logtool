## 系统要求

### 📦 打包时（开发环境）

#### 必需组件
- **操作系统**: Windows 7 或更高版本（推荐 Windows 10/11）
- **Node.js**: 16.x 或更高版本（推荐 18.x LTS）
- **npm**: 随 Node.js 自动安装（要求 8.x 或更高版本）
- **磁盘空间**: 至少 2 GB 可用空间（用于 Node.js、依赖包和打包输出）

#### 推荐配置
- **内存**: 至少 4 GB RAM
- **CPU**: 双核或更高
- **网络**: 稳定的互联网连接（首次打包需要下载 Electron 二进制文件）

#### 可选组件
- **Visual Studio Build Tools**: 如果需要编译 native 模块（如 better-sqlite3）
  - 大多数情况下不需要，electron-rebuild 会自动处理

### 🖥️ 运行环境（用户端）

#### 最低系统要求
- **操作系统**: Windows 7 或更高版本
- **架构**: 64位 (x64)
- **磁盘空间**: 至少 200 MB 可用空间（用于应用和数据）
- **内存**: 至少 2 GB RAM
- **网络**: 需要网络连接以访问日志上传服务器

#### 推荐配置
- **操作系统**: Windows 10/11 64位
- **内存**: 4 GB 或更高
- **磁盘空间**: 500 MB 或更高
- **网络**: 稳定的本地网络或互联网连接

#### 功能要求
- **文件系统访问**: 需要读取访问权限以监控指定目录
- **网络访问**: 需要访问配置中的 API 服务器
- **通知权限**: 需要通知权限以显示桌面通知（可选）

## 前置要求

### 开发环境准备

1. **安装 Node.js**
   - 访问 [Node.js 官网](https://nodejs.org/) 下载 LTS 版本
   - 推荐版本: 18.x 或 20.x LTS
   - 安装时选择 "Add to PATH" 选项

2. **验证安装**
   ```bash
   node --version    # 应该显示 v16.x 或更高
   npm --version     # 应该显示 8.x 或更高
   ```

3. **可选：安装 Visual Studio Build Tools**
   - 仅在需要编译 native 模块时安装
   - 下载地址: [Visual Studio Build Tools](https://visualstudio.microsoft.com/downloads/#build-tools-for-visual-studio-2022)
   - 安装时选择 "C++ build tools" 工作负载

### 用户端环境准备

**用户无需安装任何额外软件**，只需要：
- Windows 操作系统
- 足够的磁盘空间
- 运行打包好的 .exe 文件即可

## 快速开始

### 方式一：使用批处理脚本（最简单）⭐ 推荐

**双击运行 `build.bat` 文件**，脚本会自动：
1. 检查 Node.js 环境
2. 检查并安装依赖
3. **选择打包类型**（便携版或安装程序）
4. 清理旧的构建文件
5. 开始打包
6. 完成后自动打开 release 文件夹

#### 打包类型选择

运行 `build.bat` 时，您可以选择：

- **选项 1：便携版（Portable）**（默认）
  - 生成单个 .exe 文件
  - 无需安装，直接运行
  - 适合测试和临时使用

- **选项 2：安装程序（Installer）**
  - 生成完整的 NSIS 安装程序
  - 可创建桌面和开始菜单快捷方式
  - 可通过控制面板卸载
  - 适合正式发布和长期使用

### 方式二：使用 npm 命令

#### 1. 安装依赖

```bash
npm install
```

#### 2. 打包应用程序

**打包所有目标（推荐）**，生成安装程序和便携版：
```bash
npm run build
```

**仅打包安装程序**：
```bash
npm run build:installer
```

**仅打包 Windows 版本**：
```bash
npm run build:win
```

## 配置说明

### 自定义配置

打包时，应用会使用项目根目录的 `config.json` 作为默认配置。用户首次运行时可以修改配置。

**配置文件位置（打包后）**：
- Windows: `C:\Users\<用户名>\AppData\Roaming\日志监控客户端\config.json`

**图标文件**：
- 使用的图标位于：`src/assets/logo.ico`

### 文件打包

只打包以下文件：
- `src/**/*` - 所有源代码
- `config.json` - 配置文件
- `package.json` - 包信息

其他文件（如 `node_modules`）会自动处理。

### 自定义配置

如需修改配置，编辑 `package.json` 中的 `build` 块：

- 修改 `productName` 更改应用名称
- 修改 `appId` 更改应用标识
- 修改 `win.target` 更改打包目标
- 修改 `nsis` 选项自定义安装行为

示例：只打包便携版
```json
"win": {
  "target": [{
    "target": "portable",
    "arch": ["x64"]
  }]
}
```

## 常见问题

### 关于系统要求

#### Q: 用户是否需要安装 Node.js？
**A: 不需要！** 打包后的 .exe 文件包含所有依赖，用户直接运行即可。

#### Q: 支持 Windows 7/8/10/11 吗？
**A: 支持 Windows 7 及以上所有版本**。打包配置为 64位 (x64) 架构。

#### Q: 需要管理员权限吗？
**A: 通常不需要**，但取决于监控的目录权限：
- 监控用户目录：普通权限即可
- 监控系统目录：可能需要管理员权限

#### Q: 是否支持 32位 Windows？
**A: 目前仅支持 64位**。如需 32位支持，需要修改 `package.json` 中的 `arch` 配置。

#### Q: 需要网络连接吗？
**A: 是的**，应用需要网络连接以：
- 上传日志文件到服务器
- 进行用户认证
- 验证 token 有效期

### 关于打包

#### Q: 打包时提示缺少图标文件？
**A:** 确保 `src/assets/logo.ico` 文件存在。如不存在，需要准备一个 .ico 格式的图标文件。

#### Q: 可以修改应用名称吗？
**A:** 修改 `package.json` 中的 `build.productName` 字段。

#### Q: 如何只打包便携版？
**A:** 修改 `package.json` 中的 `build.win.target`，只保留 `portable` 目标。

#### Q: 打包的文件太大？
**A:** 可以通过调整 `files` 配置来排除不必要的文件，或者使用代码压缩工具。

#### Q: 首次打包为什么很慢？
**A:** 首次打包需要下载 Electron 二进制文件（约 100-200MB），后续打包会快很多。

#### Q: 打包时提示需要 Visual C++ Redistributable？
**A:** 这通常不是问题，打包工具会自动处理。如果编译 native 模块失败，可能需要安装 Visual Studio Build Tools。

## 故障排除

### 1. 打包失败 - 找不到图标
确保 `src/assets/logo.ico` 文件存在且为有效格式。

### 2. Native 模块编译错误
如果使用了 native 模块（如 better-sqlite3），确保已安装相应工具：
```bash
npm install --save-dev electron-rebuild
npm run rebuild
```

### 3. 打包文件过大
可以通过以下方式优化：
- 使用 `files` 配置排除不必要的文件
- 使用 `asar` 打包（electron-builder 默认启用）

### 4. 杀毒软件误报
某些杀毒软件可能误报未签名的应用程序，这是正常现象。可以：
- 将应用添加到杀毒软件白名单
- 进行代码签名（需要证书）

## 高级配置

### 代码签名（可选）

如果需要为应用程序签名，配置 `build.win.certificateFile` 和密码：

```json
"win": {
  "certificateFile": "path/to/certificate.pfx",
  "certificatePassword": "your-password"
}
```

**注意事项**：
- 需要有效的代码签名证书（.pfx 格式）
- 证书通常需要购买
- 签名后可以避免杀毒软件误报

### 自动更新（可选）

可以集成 electron-updater 实现自动更新功能：

```bash
npm install electron-updater
```

然后在主进程中配置自动更新逻辑。

### 相关命令

```bash
# 只打包安装程序
npm run build:installer

# 只打包 Windows 版本
npm run build:win

# 开发模式运行
npm run dev

# 启动应用
npm run start
```

## 注意事项

### 开发环境注意事项

1. **首次打包较慢**：需要下载 Electron 二进制文件（约 100-200 MB）
2. **文件大小**：生成的 .exe 文件约为 100-200 MB（包含 Electron 运行时）
3. **签名**：如需代码签名，需要配置有效的证书

### 用户环境注意事项

1. **杀毒软件**：某些杀毒软件可能误报未签名的应用程序，这是正常现象
   - 解决方案：将应用添加到杀毒软件白名单，或进行代码签名

2. **权限要求**：
   - 监控用户目录：普通权限即可
   - 监控系统目录：可能需要管理员权限
   - 安装到系统目录：需要管理员权限

3. **网络环境**：
   - 需要能够访问配置中的 API 服务器地址
   - 如果 API 在局域网，确保网络连通
   - 如果使用 HTTPS，确保证书有效

4. **防火墙设置**：
   - 如果企业防火墙阻止，需要添加应用或端口到白名单
   - 出站端口：取决于 API 服务器端口（通常是 3000, 443, 80）

5. **便携版 vs 安装程序**：
   - **便携版**：不需要安装，但可能需要手动创建快捷方式
   - **安装程序**：自动创建快捷方式，可以在控制面板卸载

### 依赖说明

应用依赖的 native 模块（可选）：
- **better-sqlite3**: 用于 SQLite 数据库（可选依赖，安装失败不影响核心功能）
- **7zip-bin**: 用于解压 .7z 文件
- **chokidar**: 用于文件系统监控

这些依赖在打包时会自动包含在 .exe 中。
